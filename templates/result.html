⚡ Flask API

from flask import Flask, request, jsonify
from tensorflow.keras.models import load_model
import pickle

app = Flask(__name__)
model = load_model('model.h5')
scaler = pickle.load(open('scaler.pkl', 'rb'))

@app.route('/predict', methods=['POST'])
def predict():
    data = request.json
    X = scaler.transform([data.values()])
    pred = model.predict(X)
    return jsonify({'prediction': float(pred[0])})

if __name__ == '__main__':
    app.run(debug=True, port=5000)

🚀 FastAPI

from fastapi import FastAPI
from pydantic import BaseModel
from tensorflow.keras.models import load_model
import pickle

app = FastAPI()
model = load_model('model.h5')

class Input(BaseModel):
    features: list

@app.post("/predict")
async def predict(input: Input):
    pred = model.predict([input.features])
    return {"prediction": float(pred[0])}

📊 Batch Prediction

import pandas as pd
import numpy as np
from tensorflow.keras.models import load_model
import pickle

# Load artifacts
model = load_model('model.h5')
scaler = pickle.load(open('scaler.pkl', 'rb'))

# Read and process batch
df = pd.read_csv('batch_data.csv')
X = scaler.transform(df)

# Predict
predictions = model.predict(X)
df['prediction'] = predictions

# Save results
df.to_csv('predictions.csv', index=False)

🚀 Deployment Guides

Local Development (Flask)

  1. Extract all files from ZIP
  2. Install dependencies: pip install -r requirements.txt
  3. Run server: python app.py
  4. Test at http://localhost:5000

Docker Deployment

  1. Build: docker build -t model:latest .
  2. Run: docker run -p 5000:5000 model:latest
  3. Push to registry (Docker Hub, ACR, etc.)
  4. Deploy to cloud platform

Render Deployment

  1. Create account on render.com
  2. Connect GitHub repository
  3. Add Environment: FLASK_ENV=production
  4. Deploy with one click

HuggingFace Spaces

  1. Go to huggingface.co/spaces
  2. Create new Space (Streamlit)
  3. Upload app.py and model files
  4. Share publicly with the community

Azure App Service

  1. Create Web App on Azure
  2. Deploy via GitHub Actions or ZIP upload
  3. Configure environment variables
  4. Access at custom domain

AWS / GCP / Lambda

  1. Package as Lambda function or container
  2. Upload to cloud storage
  3. Deploy with CloudFormation/Terraform
  4. Set API Gateway endpoint
